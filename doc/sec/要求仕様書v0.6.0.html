実現する機能の説明

  1. 選択した画像を移動する機能を実装する。
  2. 画像を移動した後、残りの画像でカレントディレクトリの画像表示を最大100件で更新する。

実現する機能の要求

要求番号 分類 内容
1 要求 チェックボックスをチェックした画像が存在する場合、［フォルダへ移動］ボタンをクリックするとを選択したフォルダへ移動させること。
理由 画像の整理を実行するため。
説明 この要求は、このツールの目的を実現するものとなる。
2 要求 チェックボックスををチェックした画像が存在しない場合、［フォルダへ移動］ボタンをクリックするとメッセージボックスを表示すること。
理由 チェックボックスを選択していない旨をユーザーへ伝えるため。
説明 この要求の動作はv0.5.0を踏襲する。不自然な動作である場合、新しい動作を検討する。
3 要求 移動先を選択せずに［フォルダへ移動］ボタンを押下した場合、移動先を選択していない旨をメッセージボックスで表示する。
理由 スタブで作っていた機能を流用するため。
説明 「ふるいわけ.htm」は移動先をデフォルト選択していたが、その選択先が正しいのかは判断できないので移動させる処理はやめてメッセージボックスを表示することにする。
4 要求 チェックボックスの全選択・全選択解除できるようにすること。
理由 ［フォルダへ移動］ボタンが有効に動作するタイミングに併せて機能を提供するため。
説明 すでに実装済みのボタンに機能を追加する。

画像表示と移動のシーケンス図

やったことと不足していること

  1. FileStreamとMemoryStreamの違いがよくわかっていない。このシーケンスもMemoryStreamを使うことで解決できたりしないだろうか。
  2. MemoryStreamはClose概念がないっぽいが…ストリームを閉じなくても画像を移動できるのだろうか…。
  3. v0.6.0では表示中の画像についてはストリームを開きっぱなしにしており、［フォルダへ移動］ボタンを押下すると全ストリームを閉じて再描画している。このやり方が適切なのか確認が必要と思われる。特にメモリ使用量について（→動かしてみた感じ、メモリ使用量は大丈夫そうだった。）
  4. 画像を100件表示していると、再描画にも若干時間がかかるので進捗状況の表示を行ったほうが良いと感じた。
  5. そろそろ複雑になってきたので、単体試験を作りたいものだ。
